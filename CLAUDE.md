Approx 15,150 images total. Folder structure after unzip: `<root>/<class_name>/*.jpg`. **Verify the exact top-level folder name after unzipping in Phase 0** — set `data_dir` accordingly. **Runtime:** Google Colab (T4 GPU). All paths assume `/content/` working directory.

## Critical Constraints (Don't Violate)

1. **Framework: PyTorch only.** Do not mix TensorFlow.
2. **Single-file scripts where possible.** Each phase is one runnable script. No deep package hierarchies — this is a 4-week course project, not a production library.
3. **Reproducibility.** Set seeds (`torch`, `numpy`, `random`) at the top of every training script. Use `seed=42`.
4. **Class imbalance is real.** The dataset is imbalanced (`clothes` ≈ 5325 images, `battery` ≈ 945). Always use `WeightedRandomSampler` or class-weighted loss. Never train without addressing this.
5. **Safety-critical confusions matter.** Per the proposal, batteries misclassified as metal is a fire hazard. Always log this specific cell of the confusion matrix and flag it in the report.
6. **Two model tracks must be trained:**
   - Custom CNN (baseline, from scratch)
   - Transfer learning (MobileNetV2 — primary deployment target)
   - Optional: ConvNeXt-Tiny if time permits, for the report comparison
7. **Image size: 224×224.** Standard ImageNet input — required for transfer learning models.
8. **Don't over-engineer.** No MLflow, no Docker, no FastAPI. This is "working code + results." Save metrics as CSV/JSON, save plots as PNG, save models as `.pt`.

## Phased Execution Plan

Work through these phases **in order**. Do not skip ahead. After each phase, stop and let the user verify before moving on.

### Phase 0 — Environment Setup
- Create `requirements.txt`: `torch`, `torchvision`, `timm`, `scikit-learn`, `matplotlib`, `seaborn`, `pandas`, `numpy`, `tqdm`, `pillow`, `kaggle`.
- Create `.gitignore` excluding: `outputs/checkpoints/`, `data/`, `*.pt`, `__pycache__/`, `.ipynb_checkpoints/`.
- Write `scripts/download_data.py` that uses the Kaggle API to pull `hassnainzaidi/garbage-classification` and unzips into `data/raw/`. After unzip, **list the directory tree once** to confirm the actual folder structure and print the resolved `data_dir`. Do not hardcode assumptions about the inner folder name.

### Phase 1 — Data Pipeline & EDA
- `src/data.py` must expose:
  - `get_dataloaders(data_dir, batch_size, img_size=224, val_split=0.15, test_split=0.15, num_workers=2)` returning `(train_loader, val_loader, test_loader, class_names)`.
  - Stratified split — use `sklearn.model_selection.train_test_split` on file paths with `stratify=labels`. Splits must be deterministic (seed=42).
  - Train transforms: `Resize(256) → RandomCrop(224) → RandomHorizontalFlip → RandomRotation(15) → ColorJitter(0.2,0.2,0.2) → ToTensor → Normalize(ImageNet mean/std)`.
  - Val/test transforms: `Resize(256) → CenterCrop(224) → ToTensor → Normalize`.
  - Provide a `WeightedRandomSampler` option for the train loader, weights = `1 / class_count`.
- `notebooks/01_eda.ipynb`: class distribution bar chart, sample grid (4 per class), image size distribution. Save the bar chart to `outputs/figures/class_distribution.png`.
- **Stop here. Show the class distribution to the user before training.**

### Phase 2 — Models
`src/models.py` exposes a single factory:
```python
def build_model(name: str, num_classes: int = 12, pretrained: bool = True) -> nn.Module
```
Supported `name` values:
- `"custom_cnn"` — 4 conv blocks (Conv→BN→ReLU→MaxPool), channels `[32, 64, 128, 256]`, then `AdaptiveAvgPool2d(1)` → `Dropout(0.5)` → `Linear(num_classes)`. Roughly ~1M params.
- `"mobilenet_v2"` — `torchvision.models.mobilenet_v2(weights=...)`, replace `classifier[1]` with `nn.Linear(1280, num_classes)`. Freeze backbone for first 5 epochs, then unfreeze (handled in train loop, not here).
- `"convnext_tiny"` — via `timm.create_model("convnext_tiny", pretrained=True, num_classes=12)`. Optional.

### Phase 3 — Training
`src/train.py` is the main entry point. CLI:
```bash
python src/train.py --model mobilenet_v2 --epochs 25 --batch-size 64 --lr 1e-3
```

Required behavior:
- **Optimizer:** AdamW, `weight_decay=1e-4`.
- **Scheduler:** `CosineAnnealingLR` over total epochs.
- **Loss:** `CrossEntropyLoss` with `weight=class_weights` (computed as `1 / class_freq`, normalized).
- **Mixed precision:** Use `torch.amp.autocast` and `GradScaler`. Colab T4 benefits significantly.
- **Two-stage transfer learning** (only when `--model mobilenet_v2` or `--model convnext_tiny`):
  - Epochs 1–5: freeze backbone, train classifier head only, `lr=1e-3`.
  - Epochs 6–end: unfreeze all, `lr=1e-4`.
- **Logging per epoch:** train/val loss, train/val accuracy, train/val macro-F1. Append to `outputs/metrics/{model}_history.csv`.
- **Checkpointing:** save best model by val macro-F1 to `outputs/checkpoints/{model}_best.pt`. Save final model too.
- **Early stopping:** patience=7 epochs on val macro-F1.
- Use `tqdm` for progress bars.

### Phase 4 — Evaluation
`src/evaluate.py`:
- Load best checkpoint, run on test set.
- Print and save: overall accuracy, macro-F1, weighted-F1, per-class precision/recall/F1 (CSV).
- Generate confusion matrix → `outputs/figures/{model}_confusion_matrix.png`. Use seaborn heatmap, normalize by true class (rows sum to 1), annotate with 2-decimal floats.
- **Critical safety check:** explicitly print `battery → metal` misclassification rate and `battery → trash` rate. If either exceeds 5%, log a WARNING.
- Generate training curves plot (loss + accuracy over epochs) → `outputs/figures/{model}_curves.png`.

### Phase 5 — Comparison & Report Artifacts
- Script `scripts/compare_models.py` reads all `*_history.csv` files and produces:
  - A combined bar chart of test macro-F1 per model → `outputs/figures/model_comparison.png`.
  - A markdown table dumped to `outputs/metrics/comparison.md` with columns: Model, Params (M), Test Acc, Test Macro-F1, Battery→Metal Error %.
- This table is what goes into the final report.

## Coding Conventions

- **Type hints on all function signatures.**
- **Docstrings:** one-line summary minimum for every public function. Google style if longer.
- **No magic numbers in training loops.** Pull hyperparameters from CLI args or a `config` dict at the top of the script.
- **Print blocks:** use `print(f"[{phase}] message")` so output is grep-able. Phases: `DATA`, `MODEL`, `TRAIN`, `EVAL`.
- **Error handling:** wrap data loading in try/except with a useful message about path/Kaggle credentials.
- **No global state.** Pass things explicitly. The only globals allowed are `SEED = 42` and `DEVICE = torch.device(...)`.

## What NOT to Do

- Don't write a custom data augmentation library — use `torchvision.transforms.v2`.
- Don't implement attention/transformers from scratch — use `timm` if needed.
- Don't add Weights & Biases, MLflow, TensorBoard, or any experiment tracker. CSVs are enough.
- Don't tune hyperparameters with Optuna or Ray Tune. Manual sweeps over 2-3 LRs is the budget.
- Don't write inference servers, ONNX export, or quantization unless explicitly asked. The report only needs accuracy results.
- Don't refactor working code "to make it cleaner" between phases. Move forward.
- Don't commit `data/` or `outputs/checkpoints/` — they're in `.gitignore`.
